# PR: Role Management Integration and Staff View Fixes

## Overview
This PR introduces a comprehensive role management system within the staff module, allowing administrators to define custom roles with granular navigation permissions. It also includes critical bug fixes for the staff creation views and navigation improvements.

### 1. Dynamic Role Management
- **AddRoleModal Component**: A new, modern modal that enables on-the-fly creation of custom roles.
- **Granular Permissions**: Integrated with `DASHBOARD_NAV_ITEMS` to allow selecting specific accessible menus and submenus for any created role.
- **Loophole-Proof Selection**: Implemented in `StaffForm` with state-reversion logic; if a user starts adding a role but cancels the modal, the dropdown intelligently reverts to their previous selection.

- **TypeScript Error Fix**: Resolved a critical prop mismatch in `StaffNewView` where an unnecessary `staffId` was being passed to the form component.

- **Add Staff Shortcut**: Updated the `dashboard-nav-config.ts` to replace the "Invites" placeholder with a direct "Add staff" link, streamlining the onboarding workflow for new employees.
- **Mobile Responsiveness**: Ensured all new UI elements and previous landing page sections are strictly mobile-first responsive.

## Technical Details
- Permissions are mapped to the core sidebar ID system for perfect parity between the "Add Role" UI and the actual "Sidebar" visibility.

- Used `useRef` to track selection history for non-destructive modal cancellation.
